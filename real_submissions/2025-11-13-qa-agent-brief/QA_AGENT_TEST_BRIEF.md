## EDEM Intelligence — тест-поручение для ChatGPT-агента

### 1. Контур и цели

- **Цель:** подтвердить готовность продовой версии `https://ai-chat-53bwg0tj6-moytelefonmsk-6183s-projects.vercel.app` для онбординга живых пользователей.
- **Фокус:** визуальный слой (лендинг, демо), продуктовые потоки (регистрация → welcome → чат), AI-ответы (качество, поток, ошибки), платёжные зацепки.

### 2. Среда и учётные данные

- Тестировать только на **прод-URL** выше. Dev-серверы не запускать.
- Email для регистрации — одноразовый, но рабочий (проверь почтовый ящик).
- Письма Supabase могут попасть в спам: проверить обе вкладки.
- YooKassa и крипто-платежи подключены в режиме «песочница»; оплату не проводим, проверяем наличие CTA и корректность ссылок.

### 3. Сценарии высокого приоритета

1. **Лендинг**
   - Сходу оценить визуал (Grid, состояния кнопок, адаптив 1280/1024/768/414 px).
   - Проверить ссылки: «Попробовать демо», «Войти», «Зарегистрироваться».
   - Открыть DevTools → Lighthouse (Performance + Accessibility ≥ 80? записать фактические цифры).
2. **Демо-чат без регистрации**
   - Допустимые 2 сообщения пользователя. Убедиться, что ограничение срабатывает, подсчёт ведётся.
   - Проверить, что ответы ИИ приходят, текст адекватен (содержит фирменный тон).
3. **Регистрация → Welcome → Чат**
   - Валидировать поля: слабый пароль, пароли не совпадают, дубликат email.
   - Успешный кейс: регистрация → авто-переход на `/welcome` → кнопка «Начать общение» → переход на `/chat`.
   - После подтверждения email (пройти по ссылке в письме) — убедиться, что повторный вход в `login` работает.
4. **Авторизация**
   - Ошибочный логин (неверный пароль) → корректное сообщение.
   - Прямой доступ на `/chat` без сессии → редирект на `/login`.
5. **Чат**
   - Поток SSE: ответ должен приходить частями без зависаний, не дублируется.
   - Обработка ошибок: выключить сеть в середине ответа → должна появиться toast-ошибка, UI восстанавливается.
   - Проверить лимиты: 3 быстрых сообщения подряд → нет ли rate-limit ошибки 429; если есть, зафиксировать текст.
6. **Админ-панель**
   - Войти под обычным аккаунтом → редирект на `login`.
   - Подтвердить, что разделы статистики грузятся без ошибок (если доступных данных нет — убедиться, что UI показывает пустое состояние).

### 4. UX и легальные углы

- Проверить наличие ссылок на политику/оферту (если отсутствуют — пометить в отчёте).
- Контраст текста и фона ≥ WCAG AA (использовать DevTools → Accessibility).
- Проверить, что все кнопки и формы имеют `hover/focus` состояния.
- Убедиться, что banner демо корректно сообщает об ограничении и CTA ведёт к регистрации.

### 5. Наблюдаемость и логирование

- Включить DevTools → Network, фиксировать запросы к `/api/chat`, `/api/chat/stream`, `/api/demo`.
- Отмечать задержку первого байта (TTFB) и общее время ответа.
- В консоли не должно быть красных ошибок; предупреждения записать отдельно.

### 6. Отчётность

- Использовать шаблон `TEST_REPORT_TEMPLATE.md`.
- Каждую находку маркировать по приоритету: Blocker / High / Medium / Low.
- Приложить скриншоты (если есть) и `HAR` при сетевых сбоях.
- В отдельном разделе — рекомендации по улучшению AI-ответов (тон, релевантность, скорость).

- При критическом блокере немедленно остановить тестирование и сообщить.
